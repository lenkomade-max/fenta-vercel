# Billing System — Обзор

## Гибридная модель биллинга

Fenta использует гибридную модель: **подписка + токен-метеринг**.

```
┌─────────────────────────────────────────────────┐
│         SUBSCRIPTION (Ежемесячная база)         │
├─────────────────────────────────────────────────┤
│  - Доступ к функциям                            │
│  - Включенные квоты                             │
│  - Приоритетная поддержка (высокие тарифы)     │
│  - Предсказуемая месячная стоимость            │
└─────────────────────────────────────────────────┘
                       +
┌─────────────────────────────────────────────────┐
│         TOKEN METERING (По использованию)       │
├─────────────────────────────────────────────────┤
│  - LLM токены (prompt + completion)             │
│  - KAI кредиты (генерация изображений/видео)   │
│  - Render seconds (время обработки видео)       │
│  - Storage (ГБ/месяц)                          │
└─────────────────────────────────────────────────┘
```

|--------|-------|

## Цикл биллинга

```
1-е число ─────────────────────────────── 1-е число
    │                                         │
    │  ┌─────────────────────────────────┐   │
    │  │    Подписка списывается          │   │
    ▼  └─────────────────────────────────┘   │
    │                                         │
    │  ┌─────────────────────────────────┐   │
    │  │    Использование отслеживается   │   │
    │  └─────────────────────────────────┘   │
    │                                         │
    │  ┌─────────────────────────────────┐   │
    │  │    Квоты сбрасываются            │   │
    ▼  └─────────────────────────────────┘   ▼

    Конец месяца: расчет превышения
```

1. **Списание подписки** (1-е число)
   - Базовая плата за тариф
   - Через Stripe subscription

2. **Отслеживание использования** (в течение месяца)
   - Real-time трекинг
   - Ежедневная агрегация
   - Алерты при приближении к лимиту

3. **Расчет превышения** (конец месяца)
   - Суммирование использования сверх квоты
   - Применение тарифов плана
   - Генерация счета

4. **Оплата** (автоматически)
   - Списание с карты
   - Email с чеком

## Методы оплаты

**Поддерживаются:**
- Кредитные/дебетовые карты (Stripe)
- ACH банковский перевод (только US, Enterprise)
- Wire transfer (Enterprise)
- Invoice + NET30 (Enterprise, годовой контракт)

**Не поддерживаются:**
- PayPal
- Криптовалюта
- Предоплаченные карты (в большинстве случаев)
